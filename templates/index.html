✕ Simple Security Property — No Read Up
A subject at clearance level L cannot read an object classified above L. Files above your level are not visible in the repository at all — they are hidden, not just blocked.
Example: An Intern (Level 2) cannot see Secret (Level 3) or Top Secret (Level 4) files. They simply don't appear in the file list.
✕ Star Property (★) — No Write Down
A subject at clearance level L cannot write to an object classified below L. This applies to all users including admins.
Example: A Manager (Level 4) cannot create a Public (Level 1) file. Only Level 4 or above is available as a write target.
✓ Applies to Admins Too
Admin role grants user management capabilities only. It does not override BLP access rules. An admin with Level 4 clearance follows the same read/write restrictions as any other Level 4 user.
